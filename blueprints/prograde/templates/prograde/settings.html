How PJ Stacking Works

The PJ load checker treats each column as a vertical stack on lower or upper deck. It validates total footprint, per-column height, and special nesting/axle-drop assumptions.

  • Carrier deck geometry and clearances come from Carrier Config.
  • Tongue groups, height reference, and measurement offsets drive SKU footprint and height behavior.
  • PJ SKU Overrides is the planner cheat sheet for model-specific exceptions.
How Big Tex Stacking Works

The Big Tex checker evaluates stack positions as bounded zones. It validates zone length and cumulative height per stack with warning rules for sequence-sensitive cases.

  • Carrier Config defines the trailer baseline used in the canvas and checks.
  • Big Tex Stack Constraint Config defines max length and max height by stack position.
  • Use these values to tune the prototype behavior without code changes.
