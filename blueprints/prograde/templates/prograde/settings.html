Universal Overview and How To Use

What Lives Here

Carrier Config controls trailer geometry shared by both brands. These values directly drive schematic scale, deck clearances, and validation caps.

How To Edit

Update one assumption at a time. Each edit saves immediately and flags open sessions for revalidation. Re-open active sessions before export.

Ground Pull Behavior

Ground pull does not use a fixed structural deck split. Runtime capacity is based on the first unit placed in the load.

PJ Path Overview

PJ uses SKU-driven measured dimensions with dump/gooseneck specific stacking logic. The PJ cheat sheet below is the source of truth for bed length, tongue, and stacked heights used in runtime checks.

  • Gooseneck handling: fixed neck envelope logic layered on top of SKU dimensions.
  • Dump handling: selected sidewall heights map into stacked envelope heights.
  • Measurement offsets: advanced assumptions that adjust measured footprint behavior.
Big Tex Path Overview

Big Tex uses category/model/item standards with footprint computed as bed length plus tongue. Stack constraint config remains active and feeds runtime zone length/clearance overlays and checks.
